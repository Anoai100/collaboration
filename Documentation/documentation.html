What is CSS

CSS short for Cascading Style Sheets allows developers to design webpages in ways html can't. Without css, the page is rendered by the browser using basic styles, just enough to make it visible.

CSS makes it possible to style specific elements of the html, or style a group of elements using selector which will be covered later on. CSS is easy to learn and the only requirements are:-

  • basic HTML; assigning class and id.
  • file management; will help to know how to access your css file within html

CSS can be written within the html element tag, or defined in the head element by nesting the style element. The most common and better style is to create a separate text file, saved with .css extension and then linking it with th HTML by nesting a link element in the html head. Such a file can be reused anytime by linking to different html files.

CSS Syntax

Before any styling, you should be able to target the element. We do this by refering to the element tag, id or class, it's called a selector. 

              selector {
                property: value;
              }

There are basically 3 types of selectors;

  • Element tag
  • .Class
  • #ID 

            tag {
              color: white;
            }
          
            .class {
              color: white;
            }
          
            #id {
              color: white;
            }

Keep in mind, the id must be unique to the element in the whole file. The class on the other hand can be used to group elements that might take the same styling. The element tag takes styles and applies to all elements of the same tag.

The Box Model

Elements in CSS are considered a 'box', with width and height, padding, margin, and borders. Close attention should be given to the sizing of the element as it can change unexpectedly. An element size will change once given padding, margin and border-width which will all be added to the width to give the total sizing.

This can be bad as it cause nearby elements to be pushed changing the whole size of the page or cause elements to overflow and sometimes overlap. There are special rules that can prevent this situation like display, box-sizing, overflow.

Display

As elements are considered 'boxes', they have certain display rules by default. Some elements are block like the h1 - h6, p, div, section, among others.

These elements break to a new a line, there padding and margin properties push other elements away from them and the element will extend inline to fill any space available in its container. The width and height are respected.

Other elements take inline display like span, em, strong, among others.

The element does not break to a new line, width and height are not applied. Vertical margins, padding and borders will not puch away other inline elements but horizontal ones will.

Elements also have display styling that controls how their content displays within them. By setting dispaly: Flex;, the element outer display remains but the inner display change to Flexbox.

Colors

In this module, we will not dive into the styling but simply state a few crucial points to follow in order to make your content readable while also attracting attention to the intended elements at the same time keeping the styling minimal.

It is very important to choose a background color that won't strain the readers eyes but also one that is not boring. The all the colors of other elements with be based off it. it is advised to keep the contrast between the background color and the color of element atleast 4.5:1 for normal text and 3:1 for large-text
